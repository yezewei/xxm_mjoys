# 语音转文本功能 PRD

## 一、需求背景

### 1.1 业务现状

当前系统中，AI外呼和人工外呼产生的通话记录仅保留录音文件，缺乏文字化的通话内容。业务人员在以下场景中面临效率瓶颈：

- **质检分析**：质检人员需逐条听录音进行评分，耗时长、效率低
- **会话回溯**：查找历史通话中的关键信息（如客户承诺、异议点），需反复播放录音
- **数据统计**：无法基于通话内容进行关键词、话术覆盖率等维度的量化分析

### 1.2 核心诉求

| 角色 | 诉求 |
|------|------|
| 质检人员 | 自动将录音转为文字，支持关键词搜索和在线审阅 |
| 业务管理者 | 基于文本数据进行话术分析、合规检查 |
| 运维人员 | 灵活控制识别范围和执行时间，合理分配服务器资源 |

### 1.3 目标

- 实现通话录音自动转文本，覆盖人工外呼、人机协同、API外呼三类场景
- 提供灵活的执行时间配置，平衡识别效率与系统负载
- 支持与智能质检模块联动，满足全时段识别需求

---

## 二、功能概述

| 项目 | 内容 |
|------|------|
| 功能名称 | 通话录音转文本（ASR） |
| 所属模块 | 全局服务设置 → 运营工具 |
| 功能定位 | 将通话录音自动转写为文字，支持质检分析、会话存档等场景 |

## 二、功能规则

### 2.1 数据处理范围

功能开启期间，系统自动处理以下三类通话记录：

| 记录类型 | 说明 |
|----------|------|
| 人机协同通话记录 | AI转人工后的坐席通话 |
| 人工外呼记录 | 坐席主动外呼的通话 |
| 人工通信接口外呼记录 | 通过API发起的人工外呼 |

### 2.2 识别时机

- 系统在**预设的执行时间段**内自动进行语音识别
- 执行时间段支持跨天配置（勾选"次日"）

### 2.3 关闭期间数据处理

- 功能关闭期间产生的通话记录**不会自动识别**
- 如需处理，需手动前往对应话单页面，选择记录进行"语音转文本"

### 2.4 智能质检联动

| 条件 | 识别范围 | 服务器要求 |
|------|----------|------------|
| 仅开启语音转文本 | 功能开启期间 + 执行时间段 | 使用默认服务器 |
| 同时开启智能质检 | 质检任务话单 + 上传录音（全时间段） | 需部署**独立识别服务器** |

## 三、配置项

### 3.1 功能开关

| 字段 | 类型 | 说明 |
|------|------|------|
| 语音转文本 | Switch | 开启/关闭功能 |

### 3.2 执行时间段

| 字段 | 类型 | 说明 |
|------|------|------|
| 开始时间 | TimePicker | 识别任务开始时间 |
| 结束时间 | TimePicker | 识别任务结束时间 |
| 次日 | Checkbox | 勾选后结束时间视为次日 |

**约束规则：**
- 结束时间必须大于开始时间（跨天场景除外）
- 建议设置在业务低峰期，避免影响系统性能

## 四、交互说明

1. **开启功能**：显示执行时间段配置项及功能规则说明
2. **关闭功能**：隐藏所有子配置项
3. **规则提示**：以 Alert 形式展示，始终可见（功能开启时）

## 五、数据流向

```
通话结束 → 判断功能是否开启 → 判断是否在执行时间段内 → 加入ASR识别队列 → 识别完成 → 写入话单
```

## 六、异常处理

| 场景 | 处理方式 |
|------|----------|
| 识别服务不可用 | 自动重试3次，失败后记录异常日志 |
| 录音文件损坏 | 跳过该条记录，标记为"识别失败" |
| 识别超时 | 单条录音超时阈值5分钟，超时后跳过 |
