# 从代码分析生成客户讲解稿

## 适用场景

用户要求为某个模块整理客户讲解稿/演讲稿/产品介绍文档，需要从代码和PRD中提炼功能要点和价值。

## 工作流程

### 第一步：定位模块文件

```bash
# 搜索模块相关页面文件
glob: **/*<模块关键词>*.vue
glob: **/*<模块关键词>*.md    # PRD文档

# 常见位置
src/pages/          # 页面组件
PRD/                # PRD文档
docs/               # 已有文档
```

### 第二步：读取关键文件

按优先级读取：
1. **主页面组件**（如 QualityWorkbench.vue、CustomerManagement.vue）
2. **子页面/详情页**（如 QualityDetailModal.vue、CustomerDetail.vue）
3. **PRD文档**（了解产品定位和功能边界）
4. **关联组件**（弹窗、抽屉等交互组件）

### 第三步：提炼功能要点

从代码中提取：
- **页面布局结构**（左右分栏、Tab切换、卡片布局等）
- **核心交互功能**（搜索、筛选、表格、弹窗、表单）
- **数据展示方式**（状态标签、进度条、评分颜色编码等）
- **特色功能点**（高级搜索、批量操作、自动追加等）

### 第四步：生成文档

输出到 `docs/` 目录，文件名格式：`<模块名>模块讲解稿.md`

## 文档结构模板

```markdown
# 象小秘<模块名> - 客户讲解稿

---

## 开场

<1-2句话引出传统痛点，点明本模块的解决方案>

---

## 核心价值

| 维度 | 传统模式 | 象小秘<模块名> |
|------|---------|--------------|
| <维度1> | <传统做法> | **<我们的优势>** |
| <维度2> | <传统做法> | **<我们的优势>** |

---

## 模块一：<子模块名>

**功能要点：**
- <功能点1>
- <功能点2>
- <功能点3>

**价值：** <一句话总结该模块的核心价值>

---

## 模块二：<子模块名>

（同上结构）

---

## 优势总结

1. **<优势1>**：<具体说明>
2. **<优势2>**：<具体说明>
3. **<优势3>**：<具体说明>

---

## 应用价值

| 指标 | 效果 |
|------|------|
| <指标1> | **<量化效果>** |
| <指标2> | **<量化效果>** |

---

*文档版本：v1.0 | 更新日期：YYYY年M月D日*
```

## 写作风格要求

1. **面向客户**：避免技术术语，用业务语言描述
2. **突出价值**：每个模块都要说明"对客户有什么好处"
3. **对比呈现**：用表格对比传统模式 vs 我们的优势
4. **量化效果**：尽量给出具体数字（如效率提升10倍+）
5. **简洁精炼**：每个模块只列要点，不展开细节

## 注意事项

- 不要暴露技术实现细节（如Vue组件名、接口路径等）
- 功能描述要从业务角度出发，而非开发者视角
- 价值点要与客户痛点对应，形成闭环
- 输出文件放在 `docs/` 目录下
