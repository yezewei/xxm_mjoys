# 质检任务功能改动整理

## 文档信息

| 项目 | 内容 |
|------|------|
| 整理日期 | 2026-06-30 |
| 涉及页面 | 质检任务创建、质检任务列表、质检任务详情 |
| 改动范围 | 抽检规则、数据追加、列表展示、详情查看 |

---

## 一、改动背景

### 1.1 业务痛点

| 痛点 | 具体表现 | 影响 |
|------|----------|------|
| **质检范围不灵活** | 仅支持全量质检，无法按需抽检 | 数据量大时质检效率低，无法聚焦重点数据 |
| **数据追加缺失** | 任务创建后无法自动获取新增数据 | 需要人工定期导出新数据，操作繁琐 |
| **列表信息不全** | 缺少质检范围和数据追加状态 | 无法快速了解任务配置，需逐个点击查看 |
| **详情查看不便** | 无法查看任务的数据配置详情 | 需要回到创建页面或询问配置人员 |

### 1.2 用户反馈

- "每次质检都要把所有数据都检一遍，太浪费时间了"
- "新增的通话记录需要手动加入质检任务，容易遗漏"
- "列表里看不出哪些任务开了自动追加，哪些没开"
- "想看看这个任务的筛选条件是什么，要找半天"

---

## 二、改动目标

### 2.1 核心目标

| 目标 | 说明 | 预期效果 |
|------|------|----------|
| **提升质检效率** | 支持抽检模式，聚焦重点数据 | 质检效率提升50%以上 |
| **减少人工操作** | 自动追加新数据，减少手动导入 | 每日节省30分钟人工操作 |
| **信息透明化** | 列表展示关键配置信息 | 快速了解任务状态，减少沟通成本 |
| **配置可追溯** | 详情页查看数据配置 | 配置一目了然，无需翻找 |

### 2.2 具体指标

| 指标 | 改动前 | 改动后 |
|------|--------|--------|
| 查看任务配置时间 | 2-3分钟（需打开详情或询问） | 5秒（列表直接显示） |
| 新增数据处理 | 人工导入（每日） | 自动追加（每日定时） |
| 抽检支持 | 不支持 | 支持3种分配方式 |
| 配置查看 | 需返回创建页 | 详情页直接查看 |

## 三、改动详情

### 3.1 抽检规则功能

#### 3.1.1 功能说明

| 功能点 | 说明 |
|--------|------|
| **质检范围选择** | 新增"全量质检"和"部分抽检"两种模式 |
| **抽检数量配置** | 支持按总量、按比例、按人均三种方式 |
| **抽样方式** | 支持平均分配、按比例分配 |
| **抽样维度** | 支持按场景、按坐席、按通话时长 |

#### 3.1.2 配置项

```
质检范围选择
├── 全量质检（默认）
│   └── 条件表达式配置
└── 部分抽检
    ├── 抽检数量：_____ 条
    ├── 分配方式：按总量 / 按比例 / 按人均
    ├── 抽样方式：平均分配 / 按比例分配
    └── 抽样维度：按场景 / 按坐席 / 按通话时长
```

#### 3.1.3 计算逻辑

| 分配方式 | 计算公式 | 示例 |
|----------|----------|------|
| 按总量 | 抽检数量 = 用户输入值 | 输入100 → 抽检100条 |
| 按比例 | 抽检数量 = 总数 × 百分比 / 100 | 总数1000，比例10% → 100条 |
| 按人均 | 人均数量 = 抽检数量 / 质检员人数 | 100条，5人 → 每人20条 |

---

### 3.2 数据追加功能

#### 3.2.1 功能说明

| 功能点 | 说明 |
|--------|------|
| **自动追加开关** | 开启后系统自动查询并追加新数据 |
| **执行时间配置** | 设置每天的执行时间（如08:00） |
| **查询时间范围** | 配置查询的时间段（当天/昨天的指定时间） |
| **条件筛选** | 按配置的条件表达式筛选数据 |

#### 3.2.2 配置项

```
自动追加新数据
├── 开关：开启 / 关闭
├── 执行时间：每天 [08:00] 点
└── 查询时间范围：[昨天] [00:00] ~ [昨天] [23:59]
```

#### 3.2.3 执行流程

```
每天到达执行时间（如08:00）
    ↓
查询配置的时间范围内的数据
    ↓
筛选满足条件表达式的数据
    ↓
对筛选结果进行去重（如开启）
    ↓
将新数据追加到质检任务
    ↓
更新任务统计数据
```

#### 3.2.4 状态影响

| 任务状态 | 自动追加行为 |
|----------|--------------|
| 进行中 | 正常执行自动追加 |
| 已暂停 | 暂停自动追加 |
| 已完成 | 停止自动追加 |
