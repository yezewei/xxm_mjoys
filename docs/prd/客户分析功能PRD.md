# 客户分析功能 PRD

## 一、功能概述

客户分析模块用于查看系统中不同客群之间的交叉指标分析报告，支持多维度自由组合分析、指标体系管理和定时报告自动生成与分发。

**核心价值**：通过多维度交叉分析，深入洞察客户群体特征，为精准营销和经营决策提供数据支撑。

---

## 二、功能架构

```
客户分析
├── 交叉分析（核心分析能力）
│   ├── 新建分析
│   ├── 分析历史
│   └── 分析详情
├── 指标管理（分析基础）
│   ├── 指标列表
│   ├── 指标分组
│   └── 指标配置
└── 定时报告（结果输出）
    ├── 报告模板
    ├── 定时任务
    └── 报告仓库
```

## 三、模块 1：客户交叉分析能力

### 3.1 功能描述

支持多维度自由组合交叉分析，可对客户群体进行深度洞察。

### 3.2 分析维度

#### 3.2.1 客户基础属性维度

| 维度类型 | 维度项 | 说明 |
|---------|--------|------|
| 人口属性 | 年龄 | 支持区间筛选（18-25、26-35、36-45、46-55、55+） |
| 人口属性 | 性别 | 男、女、未知 |
| 人口属性 | 职业 | 按职业分类枚举 |
| 人口属性 | 地域 | 省、市、区三级联动 |
| 人口属性 | 客户等级 | VIP、高价值、潜力、普通、流失风险 |
| 账户属性 | 开户时间 | 支持时间范围筛选 |
| 账户属性 | 渠道 | 线上、线下、手机银行、网银等 |
| 账户属性 | 产品签约状态 | 已签约、未签约 |
| 账户属性 | 客户归属网点 | 按网点筛选 |
| 账户属性 | 客户经理 | 按客户经理筛选 |

#### 3.2.2 业务行为维度

| 维度类型 | 维度项 | 说明 |
|---------|--------|------|
| 交易行为 | 交易频次 | 按区间筛选（0次、1-5次、6-10次、10+次） |
| 交易行为 | 交易金额 | 按区间筛选（0-1万、1-10万、10-50万、50万+） |
| 交易行为 | 交易时段 | 工作日、周末、节假日 |
| 交易行为 | 业务类型 | 存款、贷款、理财、信用卡 |
| 运营行为 | 活跃天数 | 按区间筛选（0天、1-7天、8-15天、15+天） |
| 运营行为 | 登录频次 | 按区间筛选 |
| 运营行为 | 客服进线 | 有、无 |
| 运营行为 | AI外呼接通 | 已接通、未接通 |
| 运营行为 | AI外呼意向 | 高意向、中意向、低意向、无意向 |

#### 3.2.3 价值分层维度

| 维度类型 | 维度项 | 说明 |
|---------|--------|------|
| 资产规模 | AUM资产规模 | 按区间筛选（0-5万、5-20万、20-50万、50-100万、100万+） |
| 资产规模 | 月均存款 | 按区间筛选 |
| 经营收益 | 贡献收益 | 按区间筛选 |
| 风险等级 | 风险等级 | 高风险、中风险、低风险、无风险 |
| 流失预警 | 流失预警标签 | 高流失风险、中流失风险、低流失风险 |

#### 3.2.4 标签维度

| 维度类型 | 维度项 | 说明 |
|---------|--------|------|
| 营销标签 | 营销标签 | 自定义标签枚举 |
| 客户类型 | 存量/新客 | 存量客户、新客户 |
| 睡眠客户 | 睡眠客户 | 是、否 |
| 潜力客户 | 高潜力客户 | 是、否 |
| 逾期客户 | 逾期客户 | 是、否 |

### 3.3 交叉分析功能能力

#### 3.3.1 多维度自由组合交叉

- **维度数量**：支持 2 维、3 维及多维度交叉
- **组合示例**：
  - 地域 × 客户等级 × AUM
  - 渠道 × 产品 × 活跃度
  - 年龄 × 性别 × 交易频次
- **维度筛选**：
  - 等值筛选：精确匹配
  - 区间筛选：数值范围
  - 枚举多选：多选筛选
  - 时间范围：日期区间

#### 3.3.2 分析结果可视化

| 展示形式 | 说明 |
|---------|------|
| 表格明细 | 原始数据表格展示 |
| 透视表 | 多维度交叉透视表 |
| 占比 | 各维度占比分析 |
| 环比/同比 | 时间维度对比分析 |
| 分层汇总 | 按维度分层汇总 |

#### 3.3.3 数据下钻能力

- 汇总数据可点击穿透查看底层客户明细清单
- 支持导出明细（Excel、CSV格式）
- 下钻层级：汇总 → 分组 → 明细

#### 3.3.4 数据过滤与排除规则

- 剔除测试客户
- 剔除销户客户
- 剔除无效数据
- 自定义过滤条件

### 3.4 页面功能

#### 3.4.1 交叉分析列表页

**功能说明**：展示所有交叉分析任务，支持新建、查看、编辑、删除操作。

**页面元素**：
- 搜索区域：分析名称、创建人、创建时间、状态
- 操作按钮：新建分析
- 数据表格：分析名称、维度组合、使用指标、创建人、创建时间、状态、操作

**表格字段**：

| 字段名 | 字段标识 | 类型 | 说明 |
|--------|---------|------|------|
| 分析名称 | analysisName | string | 分析任务名称 |
| 维度组合 | dimensions | string | 使用的维度组合 |
| 使用指标 | indicators | string | 使用的指标列表 |
| 创建人 | creator | string | 创建人姓名 |
| 创建时间 | createTime | datetime | 创建时间 |
| 状态 | status | enum | 草稿/已保存/已发布 |
| 操作 | action | - | 查看、编辑、删除、导出 |

**操作功能**：
- **新建分析**：打开新建分析弹窗/页面
- **查看**：查看分析详情和结果
- **编辑**：编辑分析配置
- **删除**：删除分析任务
- **导出**：导出分析结果

#### 3.4.2 新建分析弹窗

**步骤流程**：
1. **选择维度**：选择交叉分析的维度组合
2. **选择指标**：选择需要分析的指标
3. **设置筛选**：设置数据筛选条件
4. **预览结果**：预览分析结果

**表单字段**：

| 字段名 | 字段标识 | 类型 | 必填 | 说明 |
|--------|---------|------|------|------|
| 分析名称 | analysisName | string | 是 | 分析任务名称 |
| 分析描述 | description | string | 否 | 分析任务描述 |
| 交叉维度 | dimensions | array | 是 | 选择2-5个维度 |
| 分析指标 | indicators | array | 是 | 选择1-10个指标 |
| 筛选条件 | filters | array | 否 | 数据筛选条件 |
| 排序规则 | sortRule | object | 否 | 结果排序规则 |

#### 3.4.3 分析详情页

**页面布局**：
- 顶部：分析基本信息
- 中部：分析结果展示区域
  - 透视表视图
  - 图表视图
  - 明细表格视图
- 底部：操作按钮（导出、刷新、编辑）

**展示内容**：
- 交叉分析透视表
- 占比分析图表
- 环比/同比分析
- 数据明细表格
- 数据下钻功能

## 四、模块 2：交叉分析指标体系管理

### 4.1 功能描述

管理交叉分析使用的指标，支持指标的新增、编辑、停用、分组和权限控制。

### 4.2 指标分类

#### 4.2.1 规模类指标

| 指标名称 | 指标编码 | 计算逻辑 | 数据来源 |
|---------|---------|---------|---------|
| 客户总量 | customer_total | 统计所有客户数量 | 客户表 |
| 有效客户数 | customer_valid | 统计有效客户数量 | 客户表（状态=有效） |
| 存量客户数 | customer_existing | 统计存量客户数量 | 客户表（类型=存量） |
| 新增客户数 | customer_new | 统计新增客户数量 | 客户表（创建时间=当期） |
| 睡眠客户数 | customer_sleep | 统计睡眠客户数量 | 客户表（活跃天数<阈值） |

#### 4.2.2 资产价值类指标

| 指标名称 | 指标编码 | 计算逻辑 | 数据来源 |
|---------|---------|---------|---------|
| 平均AUM | aum_avg | SUM(AUM)/COUNT(客户) | 资产表 |
| 总AUM | aum_total | SUM(AUM) | 资产表 |
| 户均资产 | asset_avg | SUM(总资产)/COUNT(客户) | 资产表 |
| 理财保有量 | wealth_balance | SUM(理财余额) | 理财表 |
| 贷款余额 | loan_balance | SUM(贷款余额) | 贷款表 |

#### 4.2.3 行为效率类指标

| 指标名称 | 指标编码 | 计算逻辑 | 数据来源 |
|---------|---------|---------|---------|
| 月活跃率 | active_rate | 月活跃客户数/总客户数 | 行为表 |
| 交易笔数 | transaction_count | SUM(交易笔数) | 交易表 |
| 户均交易金额 | transaction_avg | SUM(交易金额)/COUNT(客户) | 交易表 |
| 接通率 | answer_rate | 接通次数/拨打次数 | 外呼表 |
| 意向客户占比 | interest_rate | 意向客户数/总客户数 | 外呼表 |

#### 4.2.4 经营收益类指标

| 指标名称 | 指标编码 | 计算逻辑 | 数据来源 |
|---------|---------|---------|---------|
| 人均贡献 | contribution_avg | SUM(贡献)/COUNT(客户) | 收益表 |
| 中间业务收入 | intermediate_income | SUM(中间业务收入) | 收益表 |
| 存款留存率 | deposit_retention | 期末存款/期初存款 | 存款表 |

#### 4.2.5 对比类指标

| 指标名称 | 指标编码 | 计算逻辑 | 数据来源 |
|---------|---------|---------|---------|
| 同比 | yoy | (本期-去年同期)/去年同期 | 计算得出 |
| 环比 | mom | (本期-上期)/上期 | 计算得出 |
| 占比 | ratio | 部分/整体 | 计算得出 |
| 渗透率 | penetration | 已签约客户/总客户 | 计算得出 |
| 增长率 | growth_rate | (本期-上期)/上期 | 计算得出 |

### 4.3 指标配置功能

#### 4.3.1 指标维护

- **新增指标**：定义指标名称、编码、计算逻辑、数据来源
- **编辑指标**：修改指标配置
- **停用指标**：停用不需要的指标
- **指标预览**：预览指标计算结果

#### 4.3.2 指标分组

- **分组管理**：新增、编辑、删除分组
- **分组示例**：
  - 存款指标组
  - 信贷指标组
  - 客户活跃指标组
  - 资产价值指标组

#### 4.3.3 指标权限

- **角色权限**：不同角色可见不同指标
- **数据脱敏**：敏感资产数据脱敏展示
- **权限配置**：按角色配置指标可见性

#### 4.3.4 指标校验

- **口径统一**：确保同一指标计算逻辑一致
- **数据校验**：校验指标数据完整性
- **异常检测**：检测指标异常波动

### 4.4 页面功能

#### 4.4.1 指标管理列表页

**功能说明**：展示所有指标，支持新增、编辑、停用、删除操作。

- 操作按钮：新增指标、指标分组管理
- 数据表格：指标名称、指标编码、指标分组、计算逻辑、数据来源、状态、操作

| 指标编码 | indicatorCode | string | 指标唯一编码 |

| 状态 | status | enum | 启用/停用 |
| 创建时间 | createTime | datetime | 创建时间 |
| 操作 | action | - | 编辑、停用、删除 |

#### 4.4.2 新增/编辑指标弹窗

**表单字段**：

| 字段名 | 字段标识 | 类型 | 必填 | 说明 |
|--------|---------|------|------|------|

## 五、模块 3：定时报告生成能力

### 5.1 功能描述

支持配置定时报告模板，自动生成交叉分析报告并分发给指定接收人。

### 5.2 报告模板配置

#### 5.2.1 模板绑定

- 每个模板绑定一套交叉维度 + 选定指标
- 示例：《月度各网点客户等级×AUM交叉分析报告》

#### 5.2.2 模板自定义

- **表头配置**：自定义表头标题
- **维度组合**：选择交叉维度
- **展示指标**：选择展示的指标
- **排序规则**：配置排序字段和方向
- **汇总行**：配置汇总行显示

#### 5.2.3 图文混合

- 交叉透视表
- 占比图表（饼图、柱状图）
- 趋势图表（折线图）

#### 5.2.4 报告预览

- 配置后实时预览交叉分析结果
- 校验数据正确性
- 支持调整配置后重新预览

### 5.3 定时调度规则

#### 5.3.1 执行周期

| 周期类型 | 说明 | 示例 |
|---------|------|------|
| 每日 | 每天执行 | 每天凌晨2点 |
| 每周 | 每周执行 | 每周一凌晨3点 |
| 每月 | 每月执行 | 每月5号凌晨4点 |
| 每季度 | 每季度执行 | 每季度首月1号 |
| 自定义 | 自定义Cron表达式 | 0 0 2 5 * ? |

#### 5.3.2 时间触发

- 指定凌晨/非业务高峰执行
- 减轻数据库查询压力
- 支持配置执行时间窗口

#### 5.3.3 重跑机制

- 定时任务失败支持手动重跑
- 失败重试告警
- 重试次数配置

### 5.4 报告输出与分发

#### 5.4.1 文件格式

| 格式 | 说明 |
|------|------|
| Excel | 标准Excel报表，含完整交叉分析透视数据 |
| PDF | PDF格式报表，含图表和表格 |

#### 5.4.2 分发渠道

| 渠道 | 说明 |
|------|------|
| 邮件推送 | 发送至接收人邮箱 |
| 系统站内消息 | 系统内消息通知 |
| 后台报告仓库 | 存档至报告仓库 |

#### 5.4.3 接收人配置

- 按岗位/部门配置接收名单
- 支持抄送配置
- 权限隔离：不同接收人看到不同数据范围

#### 5.4.4 历史存档

- 所有定时生成报告持久化存储
- 支持按时间检索历史报表
- 支持下载历史报表

### 5.5 页面功能

#### 5.5.1 报告模板列表页

**功能说明**：展示所有报告模板，支持新增、编辑、删除、预览操作。

**页面元素**：
- 搜索区域：模板名称、创建人、状态
- 操作按钮：新增模板
- 数据表格：模板名称、维度组合、使用指标、执行周期、状态、操作

**表格字段**：

| 字段名 | 字段标识 | 类型 | 说明 |
|--------|---------|------|------|
| 模板名称 | templateName | string | 模板名称 |
| 维度组合 | dimensions | string | 使用的维度组合 |
| 使用指标 | indicators | string | 使用的指标列表 |
| 执行周期 | scheduleType | enum | 每日/每周/每月/每季度 |
| 接收人 | receivers | string | 接收人列表 |
| 状态 | status | enum | 启用/停用 |
| 最后执行时间 | lastExecuteTime | datetime | 最后一次执行时间 |
| 操作 | action | - | 编辑、删除、预览、立即执行 |

#### 5.5.2 新增/编辑模板弹窗

**步骤流程**：
1. **基础信息**：填写模板名称、描述
2. **选择维度**：选择交叉维度
3. **选择指标**：选择分析指标
4. **配置报告**：配置表头、排序、汇总
5. **定时设置**：设置执行周期和时间
6. **分发设置**：配置接收人和分发渠道

**表单字段**：

| 字段名 | 字段标识 | 类型 | 必填 | 说明 |
|--------|---------|------|------|------|
| 模板名称 | templateName | string | 是 | 模板名称 |
| 模板描述 | description | string | 否 | 模板描述 |
| 交叉维度 | dimensions | array | 是 | 选择2-5个维度 |
| 分析指标 | indicators | array | 是 | 选择1-10个指标 |
| 输出格式 | outputFormat | select | 是 | Excel/PDF |
| 执行周期 | scheduleType | select | 是 | 每日/每周/每月/每季度 |
| 执行时间 | executeTime | time | 是 | 执行时间 |
| 执行日期 | executeDate | string | 否 | 执行日期（每月几号/每周几） |
| 接收人 | receivers | array | 是 | 接收人列表 |
| 分发渠道 | channels | array | 是 | 邮件/站内消息/报告仓库 |
| 是否启用 | isEnabled | boolean | 是 | 是否启用定时任务 |

#### 5.5.3 报告仓库列表页

**功能说明**：展示所有生成的报告，支持查看、下载、删除操作。

**页面元素**：
- 搜索区域：报告名称、模板名称、生成时间、状态
- 数据表格：报告名称、模板名称、生成时间、文件格式、文件大小、状态、操作

**表格字段**：

| 字段名 | 字段标识 | 类型 | 说明 |
|--------|---------|------|------|
| 报告名称 | reportName | string | 报告名称 |
| 模板名称 | templateName | string | 关联的模板名称 |
| 生成时间 | generateTime | datetime | 报告生成时间 |
| 文件格式 | fileFormat | enum | Excel/PDF |
| 文件大小 | fileSize | string | 文件大小 |
| 状态 | status | enum | 生成中/已完成/失败 |
| 操作 | action | - | 查看、下载、删除 |

## 六、非功能性需求

### 6.1 性能需求

- 多维度大基数客户交叉查询响应时间 < 5秒
- 百万级客户数据稳定计算
- 支持异步计算，避免页面超时

### 6.2 权限需求

| 功能 | 权限 | 说明 |
|------|------|------|
| 交叉分析查看 | analysis:view | 查看交叉分析结果 |
| 交叉分析编辑 | analysis:edit | 编辑交叉分析配置 |
| 指标管理查看 | indicator:view | 查看指标列表 |
| 指标管理编辑 | indicator:edit | 编辑指标配置 |
| 定时任务查看 | report:view | 查看定时任务 |
| 定时任务编辑 | report:edit | 编辑定时任务配置 |
| 报告下载 | report:download | 下载报告文件 |

### 6.3 数据溯源

- 报表每个指标支持查看数据来源
- 支持查看计算口径
- 支持查看统计时间范围

### 6.4 告警需求

- 定时任务执行失败告警
- 数据缺失告警
- 指标异常波动告警
- 告警渠道：邮件、站内消息、短信

### 6.5 导出能力

- 交叉分析临时查询支持导出
- 定时报告支持导出
- 导出格式：Excel、CSV、PDF
- 导出范围：汇总数据、明细数据

## 七、业务流转链路

```
1. 后台维护统一业务指标
   ↓
2. 配置交叉分析模板（选定维度 + 指标）
   ↓
3. 设置定时执行周期与接收人
   ↓
4. 系统定时自动做多维度交叉运算
   ↓
5. 生成标准化报表文件
   ↓
6. 自动推送 + 存档备查
```
