## 一、背景与目标

### 1.1 背景
当前「系统设置 - 人机协同配置 - 转人工条件」仅有 3 个选项（接通自动转接、听完开场白自动转接、触发话术/QA 转接），存在以下问题：
- 转人工模式覆盖场景不足，缺少「呼通即转 + AI 同时接待」等复合模式
- 各模式的触发逻辑和差异化说明缺失，用户难以理解选项含义
- 缺少针对不同模式的子配置项（如转接提示语、坐席介入方式），配置灵活度不够

### 1.2 优化目标
1. 将转人工条件从 3 个选项扩展为 4 个，覆盖更多业务场景
2. 为每个选项增加 Tooltip 说明，帮助用户理解各模式差异
3. 根据选中模式动态展示子配置项（转接提示语 / 坐席介入方式）
4. 优化红色提示文案，仅在必要时展示

## 二、功能需求

### 2.1 转人工条件选项调整

**位置：** 场景管理 → 系统设置 → 人机协同配置 → 转人工条件

**原选项（3 个）：**
| value | 文案 |
|-------|------|
| auto-transfer | 接通自动转接 |
| after-intro | 听完开场白自动转接 |
| trigger | 触发话术/QA 转接 |

**新选项（4 个）：**
| value | 文案 | 说明 |
|-------|------|------|
| auto-transfer | 呼通即转 | 即业界预测式外呼，AI 呼通后不播放开场白直接转接坐席 |
| auto-transfer-ai | 呼通即转(AI同时接待) | 电话接通后 AI 播放话术同时转接人工 |
| after-intro | AI播发完开场白后转接 | 客户接起电话后 AI 播放完开场白后转接坐席 |
| trigger | 触发话术/QA转接 | 流转至转人工节点或客户主动触发转人工 QA |

**展示方式：** 横向一行排列，超出自动换行

---

### 2.2 Tooltip 说明

每个选项右侧显示 `?` 帮助图标，Hover 后展示 Tooltip 说明气泡（`placement="topLeft"`，最大宽度 420px）。

#### 2.2.1 呼通即转
> 即业界预测式外呼，指 AI 呼通后不播放开场白直接转接坐席，在转人工最大等待时长期间，每间隔 10s 查找一次空闲坐席，若未找到空闲坐席，系统播放转人工坐席忙话术并自动挂断；若找到空闲坐席，坐席侧振铃，坐席振铃最大时长 30s，若坐席未接通系统播放转人工坐席忙话术并自动挂断，若坐席接通则进行人工通话。

#### 2.2.2 呼通即转(AI同时接待)
> 电话接通后 AI 播放话术同时转接人工，系统每间隔 10s 查找一次空闲坐席，若找到空闲坐席，坐席侧振铃，坐席振铃最大时长 30s，若坐席未接通，系统将不再查找新的坐席，后续由 AI 接待至通话结束，若坐席接通则停止 AI 接待进行人工通话。

#### 2.2.3 AI播发完开场白后转接
> 客户接起电话后 AI 播放完开场白后转接坐席，和呼通即转(AI同时接待)差别是转接时机不同。

#### 2.2.4 触发话术/QA转接
> 流转至转人工节点或客户主动触发转人工 QA，系统播放完成话术后触发转人工，在转人工最大等待时长期间，每间隔 10s 查找一次空闲坐席，若未找到空闲坐席，系统提示坐席忙并自动挂断；若找到空闲坐席，坐席侧振铃，坐席振铃最大时长 30s，若坐席未接通系统播放转人工坐席忙话术并自动挂断，若坐席接通则进行人工通话。

### 2.3 条件子配置项

根据选中的转人工条件，下方动态展示对应的子配置区域（浅灰背景卡片样式）。

#### 2.3.1 转接提示语（呼通即转）

**触发条件：** `transferCondition === 'auto-transfer'`

| 配置项 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| 转接提示语 | 文本输入框 | 是 | 正在为您转接人工 | 输入框宽度 400px，placeholder「请输入转接提示语」 |

#### 2.3.2 坐席介入方式（呼通即转AI同时接待 / AI播发完开场白后转接）

**触发条件：** `transferCondition === 'auto-transfer-ai' || transferCondition === 'after-intro'`

| 配置项 | 类型 | 必填 | 默认值 | 选项 |
|--------|------|------|--------|------|
| 坐席介入方式 | 单选框 | 否 | after-current-script | 见下表 |

| value | 文案 |
|-------|------|
| after-current-script | 等AI说完当前话术再接入 |
| immediate | 坐席接通后立即接管通话 |

#### 2.3.3 触发话术/QA转接

**触发条件：** `transferCondition === 'trigger'`

无额外子配置项。

---

### 2.4 红色提示文案

**展示条件：** `transferCondition !== 'trigger'` 时显示

**文案内容：**
> 注意：当前模式下，主流程转人工节点和QA转人工节点配置不生效

**样式：** 红色（`#ff4d4f`），位于子配置区域下方

## 三、界面布局示意

```
人机协同配置
├── 人机协同模式：[开关]
├── 协同方式：(●) 主动介入  (○) 转人工  (○) 主动介入 + 转人工
├── 转人工条件：
│   ├── (○) 呼通即转 ❓  (●) 呼通即转(AI同时接待) ❓  (○) AI播发完开场白后转接 ❓  (○) 触发话术/QA转接 ❓
│   │
│   └── [子配置区域]（根据选中项动态展示）
│       ├── 呼通即转 → 转接提示语：[正在为您转接人工    ]（必填）
│       ├── 呼通即转(AI同时接待) / AI播发完开场白后转接 → 坐席介入方式：(●) 等AI说完当前话术再接入  (○) 坐席接通后立即接管通话
│       └── 触发话术/QA转接 → 无额外配置
│
│   ⚠ 注意：当前模式下，主流程转人工节点和QA转人工节点配置不生效（trigger 时不显示）
│
└── 协同策略：(●) 随机空闲转接  (○) 按分发策略转接  (○) 转接给外部技能组
```

## 四、数据结构变更

### 4.1 systemSettingsForm 新增字段

```typescript
{
  // 原有
  transferCondition: 'auto-transfer' | 'auto-transfer-ai' | 'after-intro' | 'trigger'

  // 新增
  transferPrompt: string           // 转接提示语，默认 '正在为您转接人工'
  agentInterventionRule: 'after-current-script' | 'immediate'  // 坐席介入方式，默认 'after-current-script'
}
```

## 五、交互规则汇总

| 规则编号 | 描述 |
|----------|------|
| R1 | 转人工条件选项横向一行展示，超出自动换行 |
| R2 | 每个选项右侧显示 ❓ 图标，Hover 展示 Tooltip（topLeft，最大宽度 420px） |
| R3 | 选择「呼通即转」时，下方展示「转接提示语」输入框（必填，默认值：正在为您转接人工） |

| R7 | 子配置区域使用浅灰背景卡片样式（背景 #fafafa，边框 #f0f0f0）区分层级 |
