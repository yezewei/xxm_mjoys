## 一、功能概述

抽检规则模块用于配置质检任务的自动生成策略。运营人员通过创建抽检规则，系统按规则定时从外呼录音数据中抽样，自动分配给质检员完成质检任务。

**核心价值：**
- 替代人工逐条分配质检任务，实现自动化、规则化的任务生成
- 支持多种抽样策略（平均/按比例），确保质检覆盖的公平性和全面性
- 通过条件表达式精确筛选目标数据范围

## 三、新建/编辑弹窗（三步骤向导）

弹窗宽度 960px，通过步骤条引导用户分三步完成规则配置。

### 步骤 1：规则基本信息

| 字段 | 控件 | 必填 | 校验规则 | 说明 |
|------|------|------|----------|------|
| 规则名称 | Input | ✅ | 最多 50 字，显示字数统计 | 规则的唯一名称 |
| 规则说明 | Textarea | ❌ | 最多 200 字，显示字数统计，3 行 | 规则用途描述 |
| 质检方式 | Radio | ✅ | — | AI 质检 / 人工质检 |
| 质检模型 | Select | ✅ | — | 合规质检 / 营销意向 / 投诉倾向 / 服务质检 / 销售质检 |
| 分配对象 | Select（多选） | ✅ | 最多展示 5 个 Tag | 选择质检员，可多选 |

**步骤校验：** 点击「下一步」时校验 ruleName（必填）、inspectType（必填）、qualityModel（必填）、inspectors（必填且至少 1 人）。

### 步骤 2：数据抽样规则

本步骤分为三个配置区域：**采样数量**、**数据范围条件**、**抽样方式**。

#### 2.1 采样数量

| 分配方式 | 控件 | 说明 |
|----------|------|------|
| 按总数 | Radio + InputNumber（≥1，整数） | 固定抽取 N 条录音，单位"条" |
| 按比例 | Radio + InputNumber（1~100，整数） | 按数据总量的百分比抽取，单位"%" |
| 人均数量 | Radio + InputNumber（≥1，整数） | 每位质检员分配 N 条，总抽检量 = N × 质检员人数，单位"条/人" |

#### 2.2 数据范围条件

**质检对象（必填）：**
- 人机协同录音（ai_assisted）
- 人工外呼录音（manual_outbound）

**条件表达式：**

支持多组「或」条件（ConditionExpression），每组内支持多个「且」条件（ConditionItem）。

| 字段类型 | 可用操作符 |
|----------|-----------|
| 通话时间（callTime） | =、>、<、包含、介于 |
| 通话时长（callDuration） | =、>、<、包含、介于 |
| 客户意图（intent） | =、>、<、包含 |
| 坐席工号（agent） | =、>、<、包含 |
| 场景（scene） | =、>、<、包含 |

- 「介于」操作符展示两个输入框（最小值 ~ 最大值）
- 每组条件支持动态添加/删除条件项
- 支持添加多组「或」条件表达式
- 查看模式下所有条件不可编辑

#### 2.3 抽样方式

**平均采样：**
- 系统默认对每个场景、每个坐席的数据平均抽样
- 无需额外配置

**按比例采样：**
- 选择采样维度后，配置各维度的抽样比例

| 采样维度 | 说明 | 配置项 |
|----------|------|--------|
| 按场景（scene） | 各场景抽中数量符合预设比例，每个坐席尽可能相等 | 场景下拉选择 + 比例百分比，合计显示，超 100% 标红 |
| 按坐席（agent） | 各坐席抽中数量符合预设比例，每个场景尽可能相等 | 坐席下拉选择 + 比例百分比，合计显示，超 100% 标红 |
| 按通话时长（callDuration） | 每个场景、每个坐席抽中数量尽可能相等 | 时长区间（秒）+ 比例百分比，合计显示，超 100% 标红 |

**维度比例表通用规则：**
- 支持动态添加/删除行
- 底部显示比例合计，超过 100% 时数字标红提示
- 场景/坐席下拉支持搜索过滤

### 步骤 3：任务创建规则

| 字段 | 控件 | 必填 | 说明 |
|------|------|------|------|
| 执行频率 | 静态文字 | — | 固定为「每日」 |
| 执行时间 | TimePicker（HH:mm） | ✅ | 每天执行的时间点 |
| 数据时间偏移 | Radio 切换两种模式 | ✅ | 见下方详细说明 |
| 有效期 | DatePicker 起止 + 永久复选框 | ✅ | 规则生效的时间范围 |
| 启用状态 | Switch | — | 创建后是否立即启用 |

#### 数据时间偏移（详细说明）

**模式一：按天偏移**
- 输入框输入偏移天数（0~30，整数）
- 含义：抓取 T-N 日的数据生成抽检任务
- 示例：输入 1 表示抓取昨天的数据

**模式二：精确时间范围**
- 两个 TimePicker（HH:mm），选择开始时间和结束时间
- 含义：抓取当天指定时段内的外呼数据
- 示例：08:00 ~ 18:00 表示抓取当天 8 点到 18 点之间的数据

#### 策略概览卡片

步骤 3 底部展示规则概览，汇总前几步的配置信息：

| 字段 | 来源 |
|------|------|
| 规则名称 | 步骤 1 |
| 数据类型 | 步骤 2 |
| 质检方式 | 步骤 1 |
| 质检模型 | 步骤 1 |
| 采样方式 | 步骤 2 |
| 分配方式 | 步骤 2 |
| 执行频率 | 步骤 3 |
| 数据时间偏移 | 步骤 3 |
| 分配质检员 | 步骤 1 |

## 四、操作行为

### 4.1 列表操作

| 操作 | 行为 |
|------|------|
| 查看 | 打开弹窗，三步骤均可浏览，所有字段 disabled |
| 编辑 | 打开弹窗，回填已有数据，支持修改后提交 |
| 删除 | 二次确认弹窗（"确定要删除该抽检规则吗？删除后不可恢复。"），确认后删除 |
| 切换状态 | 直接 Switch 切换启用/停用，提示"已启用"/"已停用" |
| 新建 | 打开空弹窗，从步骤 1 开始填写 |

### 4.2 弹窗操作

| 按钮 | 行为 |
|------|------|
| 取消/关闭 | 关闭弹窗，不保存 |
| 上一步 | 返回上一步骤 |
| 下一步 | 校验当前步骤必填项，通过后进入下一步 |
| 完成 | 校验所有步骤，提交数据，关闭弹窗，刷新列表 |

### 4.3 查看模式差异

- 所有表单控件 disabled
- 步骤 3 底部按钮仅显示「关闭」
- 无添加/删除条件、维度行的按钮

## 七、业务规则与约束

1. **规则名称** 唯一性建议后端校验
2. **比例合计** 前端展示提示，后端提交时也应校验不超过 100%
3. **数据时间偏移** timeRange 模式下 dataEndTime 必须大于 dataStartTime
4. **有效期** endDate 必须晚于 startDate；勾选永久有效时忽略 endDate
5. **删除规则** 需二次确认，已关联质检任务的规则建议提示关联数量
6. **启用状态** 切换即时生效，停用后不再生成新任务，已生成的任务不受影响
7. **执行频率** 当前固定为每日，后续可扩展每周/每月
