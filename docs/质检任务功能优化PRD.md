## 一、背景与目标

### 1.1 背景
当前质检任务功能在创建流程、搜索筛选、数据展示等方面存在以下问题：
- 质检范围不支持灵活配置，无法满足抽检需求
- 条件表达式操作符不够丰富，缺少"介于"等常用操作
- 质检员分配机制不完善，无法支持多质检员协作
- 搜索筛选条件不完整，缺少审核员筛选
- 部分交互体验问题（placeholder显示、清除功能等）

### 1.2 优化目标
1. 支持全量质检和抽检两种质检范围模式
2. 完善条件表达式操作符，增加"介于"操作
3. 支持质检员多选及数据分配
4. 完善搜索筛选条件
5. 优化用户体验，修复交互问题

## 二、功能需求

### 2.1 质检任务创建流程优化

#### 2.1.1 新增质检范围选择（步骤2）

**位置：** 质检任务创建流程 - 步骤2

**功能描述：**
在步骤2新增"质检范围"选择，支持两种模式：

| 模式 | 说明 |
|------|------|
| 全量质检 | 对符合条件的所有数据进行质检，保留原有条件表达式配置 |
| 抽检 | 按配置的规则对部分数据进行抽检 |

**交互规则：**
- 默认选中"全量质检"
- 选择不同模式时，下方配置区域动态切换
- 切换模式时清空已配置的抽检规则

---

#### 2.1.2 全量质检模式

**功能描述：**
全量质检模式复用现有条件表达式配置功能，包括：
- 条件表达式配置（包含/不包含、大于/小于/等于/不等于、介于等）
- 已添加的表达式列表

---

#### 2.1.3 抽检模式

**功能描述：**
抽检模式包含以下配置项：

**1. 抽检条数**
- 输入框，限制输入正整数
- 表单校验：必填，正整数
- 提示：抽检条数不得大于总数据量

**2. 抽检规则列表**
- 表格展示，列：序号、规则、占比（%）、数量
- 支持新增/删除规则
- 点击"配置"打开规则配置弹窗

**3. 规则配置弹窗**
- 弹窗标题：配置表达式
- 支持配置多个表达式组
- 每个表达式组支持多个条件
- 条件配置：
  - 选择条件：开场白、关键字、通话时长、通话时间、情绪值
  - 选择操作符：包含/不包含/等于/不等于/大于/小于/大于等于/小于等于/介于
  - 输入/选择条件值（介于操作时显示两个输入框）

**4. 占比配置**
- 每条规则配置占比（%）
- 底部显示占比合计
- 占比合计必须等于100%，否则提示错误

---

#### 2.1.4 去重规则配置

**位置：** 质检范围选择区域下方（两种模式都可见）

**功能描述：**

**1. 是否去重**
- 开关控制，默认关闭

**2. 去重类型**（开启去重后显示）
- Radio选择，两个选项：
  - 全局去重：标记话术库+话术键为重复的通话，并去除重复的通话
  - 同质检模型去重：标记话术库+话术键+质检模型为重复的通话，并去除重复的通话

**3. 备注说明**
- 全局去重：标记话术库+话术键为重复的通话，并去除重复的通话
- 同质检模型去重：标记话术库+话术键+质检模型为重复的通话，并去除重复的通话

---

#### 2.1.5 条件表达式操作符增强

**功能描述：**
通话时长和通话时间字段支持"介于"操作符

**交互规则：**
- 选择"介于"操作符时，显示两个输入框（最小值 ~ 最大值）
- 通话时长：数字输入框，单位秒
- 通话时间：时间选择器
- 占位符：最小值 ~ 最大值

**支持"介于"操作的字段：**
| 字段 | 输入类型 |
|------|----------|
| 通话时长 | 数字输入框 |
| 通话时间 | 时间选择器 |

---

#### 2.1.6 质检员多选及数据分配

**功能描述：**
质检员从单选改为多选，支持多个质检员协作质检

**交互规则：**
- 质检员下拉框支持多选
- 多选后显示数据分配提示

**分配规则：**
| 质检范围 | 分配方式 |
|----------|----------|
| 全量质检 | 数据平均分配给对应账号 |
| 抽检 | 按抽检比例分配给对应账号 |

---

#### 2.1.7 抽检模式保存校验

**功能描述：**
抽检模式下点击保存时，校验数据是否满足抽检数量要求

**交互规则：**
- 点击"保存"按钮触发校验
- 校验条件表达式+去重规则后的数据条数是否满足抽检条数要求
- 满足条件：正常保存
- 不满足条件：弹窗提示"当前条件表达式+去重规则后的数据条数小于抽检条数，建议不配置去重规则或减少抽检条数"
  - 确认按钮：继续保存
  - 取消按钮：返回修改

## 三、接口变更

### 3.1 CreateTaskForm 接口

```typescript
interface CreateTaskForm {
  taskType: 'ai' | 'manual'
  taskName: string
  taskDesc: string
  qualityObject: string
  qualityModel: string
  inspector: string[]  // 从 string 改为 string[]，支持多选
  qualityScope: 'full' | 'sampling'  // 新增：质检范围
  samplingCount: number  // 新增：抽检条数
  dedupEnabled: boolean  // 新增：是否去重
  dedupType: 'global' | 'sameModel'  // 新增：去重类型
}
```

### 3.2 SamplingRule 接口（新增）

```typescript
interface SamplingRule {
  id: number

  quantity: number
  conditions: Array<{
    field: string
    operator: string
    value: string
    value2?: string  // 介于操作时的第二个值
  }>
}
```
